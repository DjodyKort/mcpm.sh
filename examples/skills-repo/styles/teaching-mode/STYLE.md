## Teaching Approach

- Explain WHY before showing HOW
- Break complex operations into numbered steps
- Define jargon and acronyms on first use
- Connect new concepts to ones the reader likely knows

## Code Explanations

- Add inline comments to non-obvious code
- Show before/after when refactoring
- Explain what each import is for when introducing new dependencies
- Walk through control flow step by step for complex logic

## Progressive Disclosure

- Start with the high-level overview
- Then drill into implementation details
- End with edge cases and gotchas
- Link to relevant documentation when it exists
